Email Attachment from CSV (Dataflow)

SpeedBee Synapse Component Reference

What This Component Does

This component collects CSV files that have been saved to disk by the Synapse pipeline and sends them as a single email attachment. When triggered, it looks back through a configured number of hours worth of saved files, merges them all into one CSV, and emails it to the specified recipients.

It can be triggered manually by an upstream signal in the pipeline, or it can run on a schedule automatically sending the merged file at a regular interval without any external trigger needed.

If no data files are found in the time window, the component skips the send and logs a message instead.

Parameters

Parameter What it does
SMTP Server
smtphost The address of the mail server used to send the email. For example, smtp.office365.com or smtp.gmail.com.
smtpport The port on the mail server to connect to. The default is 587, which is the standard port for secure STARTTLS connections.
smtpusername The username used to log in to the mail server. Usually the sender email address.
smtppassword The password for the mail server account. This is never shown in logs.
timeout How many seconds to wait when connecting to the mail server before giving up. The default is 5 seconds.
Email Content
fromaddress The email address the message will be sent from.
toaddress The recipient email address. Multiple recipients can be listed separated by commas.
subjectemail The subject line of the email.
bodyemail The body text of the email. If left blank, a default message is used. The CSV file is always included as an attachment regardless of what the body says.
File Location
savelocation The root folder on disk where the component looks for saved CSV files. This should point to the base directory used by the upstream data saving component.
savelocationcont A subfolder name appended to the save location path. Used to organise files within the root folder.
section A label identifying this data source within the folder structure. It is also used as the prefix for the attachment filename.
Scheduling
sendhours How many hours back from the current time to look when collecting CSV files. For example, a value of 1 means the component collects all files saved in the last hour. This same value is also used as the interval when periodic sending is enabled.
enableperiodic Set to 1 to have the component send automatically on a repeating schedule without needing a trigger signal. Set to 0 to only send when a trigger arrives from upstream. The repeat interval is controlled by sendhours.
