Email Export Attachment (Dataflow)

SpeedBee Synapse Component Reference

What This Component Does

This component collects live data rows arriving on PORT2 into an in-memory CSV buffer and sends the buffer as an email attachment. The send is triggered by an upstream signal on PORT1, or automatically once the buffer reaches a configured row count threshold.

The buffer is bounded — if a maximum row count is configured, the oldest rows are dropped when new ones arrive and the buffer is full. A minimum one-second gap between sends prevents burst triggers from firing multiple emails in quick succession.

The buffer is only cleared after a successful send. If the send fails, the data is discarded and the error is logged to PORT1 output.

Parameters

Parameter What it does
SMTP Server
smtphost The address of the mail server used to send the email. For example, smtp.office365.com or smtp.gmail.com.
smtpport The port on the mail server to connect to. The default is 587, which is the standard port for secure STARTTLS connections.
smtpusername The username used to log in to the mail server. Usually the sender email address.
smtppassword The password for the mail server account. This is never shown in logs.
timeout How many seconds to wait when connecting to the mail server before giving up. The default is 5 seconds.
Email Content
fromaddress The email address the message will be sent from.
toaddress The recipient email address. Multiple recipients can be listed separated by commas.
subjectemail The subject line of the email.
bodyemail The body text of the email. The CSV data is always included as an attachment regardless of what the body says.
Buffer
rowthreshold The number of buffered rows that triggers an automatic send without needing a PORT1 signal. Set to 0 to disable threshold-based sending and rely on PORT1 triggers only.
maxbufferrows The maximum number of rows to hold in memory at once. When the buffer is full, the oldest row is dropped to make room for the newest. Set to 0 for an unlimited buffer.
